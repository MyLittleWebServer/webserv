Detailed Description

Kqueue class : kevent와 FD를 관리하는 클래스입니다.

Kqueue는 kevent와 FD를 관리하는 클래스입니다. fd_set _server_fds, _client_fds, _method_fds, _cgi_fds를 갖고 있습니다. _server_fds는 서버의 fd_set입니다. _client_fds는 클라이언트의 fd_set입니다. _method_fds는 메소드의 fd_set입니다. _cgi_fds는 CGI의 fd_set입니다. _kq는 kqueue의 fd입니다. _eventsToAdd는 kevent를 추가하기 전에 임시로 저장하는 vector입니다. _eventList는 kevent를 저장하는 배열입니다. CONCURRENT_EVENTS는 kevent를 저장하는 배열의 크기입니다. _server_fds, _client_fds, _method_fds, _cgi_fds는 각각 서버, 클라이언트, 메소드, CGI의 fd_set입니다.

addEvent : kevent()를 호출하기 전에 이벤트를 _eventsToAdd에 등록

해당 이벤트는 ident로 들어온 소켓 디스크립터를 event filter, flags, fflags, data, udata로 설정합니다.

Parameters
ident: socket descriptor
filter: event filter(read/write)
flags: add, delete, enable, disable
fflags: field 유형의 flag
data: 이 필드는 필터 유형에 따라 특정한 데이터를 가집니다. 예를 들어, EVFILT_READ인 경우 data 필드는 읽을 수 있는 데이터의 바이트 수를 나타냅니다.
udata: 사용자가 임의로 설정할 수 있는 포인터입니다. 이 필드는 kevent 함수가 이벤트를 반환할 때 변경되지 않습니다.

◆ newEvents()

int Kqueue::newEvents ( void  )

newEvents 함수는 새로운 이벤트를 반환합니다.

  1. kevent() 함수를 호출하여 _kq에 등록된 이벤트가 있는지 확인합니다. 이벤트는 _eventsToAdd에 등록되며, 새로 발생한 이벤트는 _eventList에 저장됩니다. _eventToAdd.size()는 등록된 이벤트의 수를 나타내고 CONCURRENT_EVENTS는 한번에 처리할 수 있는 이벤트의 최대 수를 나타냅니다. 마지막 매개변수 NULL은 timeout에 대한 설정을 나타냅니다.
  2. kevent() 함수 호출이 성공하면, 새로운 이벤트의 수를 반환합니다. 그리고 _eventToAdd를 비웁니다.
Exceptions
kevent()error : kevent() 함수가 -1을 반환하면 에러입니다.
Returns
int : 새로운 이벤트의 수
